## Introduction

A thorough review process ensures your documentation is accurate, clear, and valuable to users. This guide outlines the steps for reviewing, testing, and maintaining high-quality documentation using Mintlify.

## Proofreading & Formatting

<Frame>
  <img src="/images/sia-logo-light.png" alt="Documentation Style Guide" />
</Frame>

<AccordionGroup>
  <Accordion icon="spell-check" title="Content Review">
    Review checklist:
    - Spelling and grammar
    - Technical accuracy
    - Clarity and conciseness
    - Consistent terminology
    
    <Tip>
      Use tools like Grammarly or Hemingway Editor for initial proofreading.
    </Tip>
  </Accordion>
  
  <Accordion icon="paintbrush" title="Formatting Standards">
    Style guidelines:
    ```markdown
    # Main Heading (H1)
    ## Section Heading (H2)
    ### Subsection (H3)
    
    - Bullet points
    - For lists
    
    1. Numbered steps
    2. For procedures
    
    `inline code`
    ```
    
    <Warning>
      Maintain consistent formatting across all documentation pages.
    </Warning>
  </Accordion>
</AccordionGroup>
